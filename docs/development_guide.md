# Development Guide

This document provides project build, configuration, and development instructions.

---

## Environment Requirements

### Required Tools

- **Android Studio**: Arctic Fox (2020.3.1) or higher, latest stable version recommended
- **JDK**: JDK 11 or higher
- **Gradle**: Project uses Gradle Wrapper, no manual installation needed
- **Android SDK**:
  - Minimum SDK: 28 (Android 9.0)
  - Target SDK: 34 (Android 14)
  - Compile SDK: 34
- **Android NDK**: r25c or higher (for libmpv native library)

### Hardware Recommendations

- **Memory**: At least 8GB RAM (16GB+ recommended)
- **Storage**: At least 10GB free space (for SDK, NDK, dependencies, and build output)
- **Test Device**: Physical device or emulator with Android 12+

## 🔧 Building the Project

### 1. Clone Repository

```bash
git clone https://github.com/azxcvn/mpv-android-anime4k.git
cd mpv-android-anime4k
```

### 2. Install Android Studio and SDK

1. Download and install [Android Studio](https://developer.android.com/studio)
2. Open Android Studio, install required components via SDK Manager:
   - Android SDK Platform 34
   - Android SDK Build-Tools 34.0.0
   - Android SDK Platform-Tools
   - Android Emulator (if using emulator)

### 3. Configure Android NDK

Project uses libmpv native library, NDK installation required:

1. Open SDK Manager in Android Studio
2. Switch to "SDK Tools" tab
3. Check "NDK (Side by side)"
4. Check "CMake" (for compiling native code)
5. Click "Apply" to install

Or manually specify NDK path in project's `local.properties` file:

```properties
ndk.dir=C\:\\Users\\YourName\\AppData\\Local\\Android\\Sdk\\ndk\\25.2.9519653
```

### 4. Configure DanDanPlay API Credentials

First copy the example configuration file:

```bash
# Windows
copy local.properties.example local.properties

# Linux/macOS
cp local.properties.example local.properties
```

Then edit `local.properties` file in project root, fill in your DanDanPlay API credentials:

```properties
# SDK path (auto-generated by Android Studio)
sdk.dir=C\:\\Users\\YourName\\AppData\\Local\\Android\\Sdk

# DanDanPlay API credentials (fill in manually)
dandanplay.appId=your_app_id_here
dandanplay.appSecret=your_app_secret_here

# NDK path (optional, if auto-detection fails)
# ndk.dir=C\:\\Users\\YourName\\AppData\\Local\\Android\\Sdk\\ndk\\25.2.9519653
```

#### 💡 How to Get DanDanPlay Credentials?

1. Go to [DanDanPlay Open Platform](https://www.dandanplay.com/)
2. Register account and apply for developer permissions
3. Create application and get AppId and AppSecret
4. Fill credentials into `local.properties` file

> **Note**: `local.properties` file is added to `.gitignore` and will not be committed to version control.

### 5. Open Project

1. Launch Android Studio
2. Select "Open" or "Open an Existing Project"
3. Navigate to cloned project directory, select root directory to open
4. Android Studio will automatically recognize Gradle project and start syncing

### 6. Sync Gradle Dependencies

When opening project for the first time, Android Studio will automatically trigger Gradle sync:

1. Wait for Gradle to download dependencies (may take several minutes)
2. If sync fails, click "Sync Project with Gradle Files" in toolbar to retry
3. Check "Build" window for sync logs

Common sync issues:

- **Network issues**: Configure Gradle to use domestic mirrors (Aliyun, Tencent Cloud, etc.)
- **NDK not found**: Check if NDK is properly installed and path is configured correctly
- **JDK version incompatible**: Ensure using JDK 11 or higher

### 7. Build Project

#### Build via Android Studio

1. Select menu `Build` → `Make Project` or press `Ctrl+F9` (Windows/Linux) / `Cmd+F9` (macOS)
2. Wait for build to complete, check "Build" window output
3. If build succeeds, will display "BUILD SUCCESSFUL"

#### Build via Command Line

```bash
# Windows
gradlew.bat assembleDebug

# Linux/macOS
./gradlew assembleDebug
```

Build output location: `app/build/outputs/apk/debug/app-debug.apk`

#### Build Release Version

```bash
# Windows
gradlew.bat assembleRelease

# Linux/macOS
./gradlew assembleRelease
```

Release version requires signing configuration, see "Signing Configuration" section below.

### 8. Run Project

#### Run on Physical Device

1. Enable "Developer Options" and "USB Debugging" on phone
2. Connect phone to computer via USB cable
3. Select device in Android Studio: device dropdown in toolbar
4. Click "Run" button (green triangle) or press `Shift+F10`

#### Run on Emulator

1. Open AVD Manager: `Tools` → `AVD Manager`
2. Create new virtual device (Pixel series recommended, API 31+)
3. Start emulator
4. Select emulator device in Android Studio
5. Click "Run" button

> **Performance Tip**: Due to high performance requirements of Anime4K upscaling algorithm, testing on physical device recommended, emulator performance may be insufficient.

## Signing Configuration (Optional)

If you need to release a Release version, signing configuration is required:

### 1. Create Keystore

```bash
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
```

### 2. Configure Signing Information

Add to `local.properties`:

```properties
storeFile=../my-release-key.jks
storePassword=your_store_password
keyAlias=my-key-alias
keyPassword=your_key_password
```

Or hardcode in `app/build.gradle` (not recommended):

```groovy
android {
    signingConfigs {
        release {
            storeFile file('../my-release-key.jks')
            storePassword 'your_store_password'
            keyAlias 'my-key-alias'
            keyPassword 'your_key_password'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

## Common Build Issues

### Issue 1: NDK Not Found

**Error Message**:
```
NDK is not configured. Download it with SDK manager.
```

**Solution**:
1. Install NDK via SDK Manager
2. Specify NDK path in `local.properties`

### Issue 2: DanDanPlay API Credentials Not Configured

**Error Message**:
```
Missing dandanplay.appId or dandanplay.appSecret in local.properties
```

**Solution**:
Configure `local.properties` file according to step 4.

### Issue 3: Gradle Sync Failed

**Error Message**:
```
Could not resolve all dependencies...
```

**Solution**:
1. Check network connection
2. Configure Gradle to use mirror sources (modify `build.gradle`)
3. Clear Gradle cache: `gradlew clean`

### Issue 4: Out of Memory

**Error Message**:
```
OutOfMemoryError: Java heap space
```

**Solution**:
Increase heap memory in `gradle.properties`:

```properties
org.gradle.jvmargs=-Xmx4096m -XX:MaxPermSize=512m -XX:+HeapDumpOnOutOfMemoryError
```

## Debugging Tips

### Using Logcat

1. Open "Logcat" window in Android Studio
2. Filter log tags: enter `tag:MPV` or `tag:Danmaku` etc.
3. View app runtime log output

### Using Debugger

1. Set breakpoints in code (click blank area next to line number)
2. Click "Debug" button (green bug icon) or press `Shift+F9`
3. App will pause at breakpoint, can view variable values and call stack

### Performance Analysis

1. Use Android Studio Profiler: `View` → `Tool Windows` → `Profiler`
2. Monitor CPU, memory, network, and power usage
3. Analyze Anime4K upscaling rendering performance bottlenecks
