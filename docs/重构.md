### 仍需迁移的页面（按优先级排序）

#### P0 — 已存在 Screen composable 的，直接集成

| 页面 | 当前启动方式 | 所需工作 |
|---|---|---|
| **LicenseScreen** | `LicenseActivity`（Intent 启动） | **已在** LicenseScreen.kt 定义 `@Composable fun LicenseScreen(onBack)`，只需在 `AppScreen` 添加路由、在 `AppNavGraph` 注册 composable，修改 `AboutScreen` 中 Intent → `navController.navigate(AppScreen.License)` |

**工作量：极小（< 10 分钟）**

#### P1 — 页面已有 Compose 内容，可从已有 Activity 中拆出

| 页面 | 当前启动方式 | 所需工作 |
|---|---|---|
| **BiliBiliDanmakuScreen** | `BiliBiliDanmakuComposeActivity` | **已存在** `BiliBiliDanmakuScreen(@Composable, 需传入 Uri?、下载状态 + 回调)`，提取到 Navigation3（参数从 Activity intent extras → NavArgs） |
| **VideoBrowserScreen** | `VideoBrowserComposeActivity` | 该 Activity 全用 Compose，需提取为 `VideoBrowserScreen` composable，改为 navigation 形式 |
| **DownloadScreen** | `DownloadActivity` | 检查是否已提取 composable，若没有则需从 Activity 中拆出 |
| **SubtitleSearchScreen** | `SubtitleSearchActivity` | 使用 `BaseActivity` + Layout XML → 需先转为 Compose 再迁移，或保持 Intent |

#### P2 — 页面逻辑复杂/强依赖 Activity 生命周期

| 页面 | 当前启动方式 | 所需工作 |
|---|---|---|
| **BiliBiliPlayScreen** | `BiliBiliPlayActivity` | 含 WebView（需 `Lifecycle`）、视频播放、输入 URL 等。提取为 composable 工作量较大 |
| **TVBrowserScreen** | `TVBrowserActivity` (companion object `start()`) | **已存在** `TVBrowserScreen(@Composable, 需 initialUrl + onWebViewCreated + onBackPressed)`，但内部有 WebView + video sniffing，需仔细处理 WebView 状态 |
| **MediaInfoScreen** | `MediaInfoActivity`（Compose Activity） | 可从 `MediaInfoActivity`（ComponentActivity）中提取 composable |
| **VideoPlayerActivity** | Intent 启动视频播放 | 是全屏视频播放器，生命周期较复杂，建议最后迁移或保持 Intent |

#### P3 — 纯工具/不需要深集成或已废弃的

| 页面 | 当前启动方式 | 所需工作 |
|---|---|---|
| **BiliBiliLoginActivity** | Intent（auth screen） | 登录页面通常 session 式，保持 Intent 或待以后 |
| **UserAgreementActivity** | Intent from `MainActivity` | 用户协议页面，可直接转为 composable |
| **RemotePlaybackLauncher.start** | `RemotePlaybackLauncher.start(context, request)` | 网络播放启动器，保持 Intent 或等 VideoPlayer 迁移后统一 |
