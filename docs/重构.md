

## VideoPlayerActivity 重构方向分析

### 一、现状评估

**核心数据**
- 代码行数：2670行（VideoPlayerActivity.kt）
- XML布局：activity_video_player.xml（复杂布局）
- XML对话框：26个（需要全部迁移到Compose）
- Manager类：11个（PlaybackEngine, ControlsManager, GestureHandler, SeriesManager, Anime4KManager, DanmakuManager等）
- 原生视图：2个（CustomMPVView, DanmakuPlayerView）
- ViewModel状态：PlayerViewModel已创建但功能极简，未充分使用

**架构现状**
- 16/17页面已完成迁移，VideoPlayerActivity是最后一个
- 其他页面已使用：Activity(极简) + Compose Screen + ViewModel + Koin模式
- 项目已完成阶段0-4（现代化构建、Koin、代码分层、部分UI Compose化、Navigation3）

### 二、核心挑战与约束

**1. 原生视图约束**
- `CustomMPVView`继承`BaseMPVView`，必须使用Android View系统
- MPV必须在Activity.onCreate中初始化（不能在View生命周期中）
- `DanmakuPlayerView`也是原生视图，需要精确的帧绘制
- **解决方案**：使用`AndroidView`包装，保持原生视图，外层用Compose

**2. 复杂状态管理**
- Activity中有40+个状态变量（isPlaying, currentPosition, duration, speed, anime4KEnabled, isLocked等）
- 状态散落在Activity、Manager、Dialog回调中
- **解决方案**：集中到PlayerViewModel的StateFlow/State对象中

**3. Manager类架构**
- 11个Manager已经是很好的职责分离（Domain层逻辑）
- 问题在于Manager与Activity耦合（通过回调和直接调用）
- **解决方案**：保留Manager类100%逻辑，改变调用方式（ViewModel调用Manager，Manager回调更新ViewModel状态）

**4. 26个XML对话框**
- 包括：速度、解码器、画面比例、音频轨道、字幕、弹幕设置、Anime4K等
- 目前使用XML + AlertDialog
- **解决方案**：使用已有的`ComposeOverlayManager`逐步迁移为Compose Dialog

### 三、重构方向（基于mpvEx实践）

**核心策略**：采用mpvEx验证过的渐进式迁移路径，结合项目已有的Manager架构

```kotlin
@Composable
fun PlayerControls(
    viewModel: PlayerViewModel,
    onBackPress: () -> Unit
) {
    val controlsShown by viewModel.controlsShown.collectAsState()
    val locked by viewModel.areControlsLocked.collectAsState()
    val paused by viewModel.paused.collectAsState()
    val position by viewModel.precisePosition.collectAsState()
    val duration by viewModel.preciseDuration.collectAsState()
    
    Box(modifier = Modifier.fillMaxSize()) {
        // 手势层
        GestureLayer(
            enabled = !locked,
            onTap = { viewModel.toggleControls() },
            onDoubleTap = { viewModel.togglePlayPause() },
            onVerticalDrag = { delta -> viewModel.handleVerticalGesture(delta) },
            onHorizontalDrag = { delta -> viewModel.handleSeekGesture(delta) }
        )
        
        // 加载/暂停指示器
        PlayerIndicators(viewModel)
        
        // 控制面板
        AnimatedVisibility(
            visible = controlsShown && !locked,
            enter = fadeIn(),
            exit = fadeOut()
        ) {
            Column(modifier = Modifier.fillMaxSize()) {
                // 顶部面板
                TopControlPanel(viewModel, onBackPress)
                
                Spacer(modifier = Modifier.weight(1f))
                
                // 底部面板
                BottomControlPanel(viewModel)
            }
        }
        
        // 锁定按钮
        LockButton(
            locked = locked,
            onToggle = { viewModel.toggleLock() }
        )
        
        // Sheet/Dialog
        PlayerSheets(viewModel)
    }
}
```

    }

### 四、技术要点

**1. 原生视图生命周期管理**
- MPV初始化必须在Activity.onCreate中完成
- 使用`DisposableEffect`管理原生视图生命周期
- 在ViewModel.onCleared()中清理资源

**2. 状态同步**
- 播放进度用LaunchedEffect轮询或MPV回调更新
- 弹幕与视频同步通过DanmakuManager.setPlaybackEngine()
- 所有Manager回调统一更新ViewModel的StateFlow

**3. 手势冲突处理**
- Compose的手势检测与原生视图手势需要协调
- 使用`Modifier.pointerInput`包装整个播放区域
- 锁定状态下禁用手势

**4. 性能优化**
- 避免频繁重组：用`derivedStateOf`、`remember`缓存计算
- 进度更新用`LaunchedEffect`而非StateFlow频繁emit
- 大列表（视频列表、字幕轨道）使用`LazyColumn`

### 六、风险与缓解

**风险1：原生视图与Compose集成问题**
- 缓解：参考官方AndroidView文档，使用`remember`缓存视图实例
- 测试：优先实现MPV显示，确保播放正常

**风险2：性能下降**
- 缓解：进度更新频率控制在500ms，使用`derivedStateOf`减少重组
- 测试：对比重构前后CPU/内存占用

**风险3：手势响应变慢**

- 测试：双击、滑动、长按灵敏度测试

**风险4：状态丢失**
- 缓解：使用`rememberSaveable`保存关键状态，ViewModel自动保存
- 测试：旋转屏幕、后台返回等场景

```kotlin
// 输出格式从 DanmakuContext 改为 List<TextData>
object BiliDanmakuLoader {
    fun parse(xmlPath: String): List<TextData> {
        val dataList = mutableListOf<TextData>()
        // 解析B站XML
        xml.forEachElement("d") { element ->
            val p = element.getAttribute("p").split(",")
            val showTime = p[0].toFloat() * 1000  // 秒转毫秒
            val type = p[1].toInt()  // 1=滚动 5=顶部 4=底部
            val color = p[3].toLong()
            val text = element.textContent
            
            dataList.add(TextData().apply {
                this.text = text
                this.showAtTime = showTime.toLong()
                this.layerType = when(type) {
                    1 -> LayerType.SCROLL
                    5 -> LayerType.TOP_CENTER
                    4 -> LayerType.BOTTOM_CENTER
                    else -> LayerType.SCROLL
                }
                this.textColor = color.toInt()
            })
        }
        return dataList
    }
}
```
